Recommended metrics

Discrimination: ROC AUC (with 95% CI via bootstrap or DeLong), PR AUC (important for imbalanced data).
Threshold metrics: sensitivity (recall), specificity, PPV, NPV, F1 at clinically-relevant thresholds.
Calibration: calibration plot, Brier score, calibration-in-the-large, calibration slope, Hosmer–Lemeshow (with caution).
Clinical utility: decision curve analysis (net benefit) and number-needed-to-treat analogues.
Robustness/fairness: per-site performance, subgroup (age/sex) performance, worst-site performance, stddev across sites.
Federated-specific: global vs per-client performance, convergence (loss / metric vs rounds), communication cost (bytes, model params sent), rounds-to-target-performance, per-round time, client computation cost, fraction of clients used per round.
Privacy/efficiency: model accuracy vs DP epsilon (if private FL), secure aggregation overhead, and added latency.

------------------------------------------------------------------------------------------------------
Baselines and experiments

Centralized upper-bound: train same model on pooled data (if allowed).
Local lower-bound: train per-client models on local data only.
Federated variants: FedAvg, FedProx, FedAvg with personalization, different client sampling rates, and DP-FedAvg.
Heuristic baseline: simple logistic regression or clinical score (e.g., BMI+age rule).
External/temporal validation: hold out one site (leave-one-site-out) and temporal split.

------------------------------------------------------------------------------------------------------
Experimental protocol

Use identical model architecture/hyperparams across settings where applicable.
Hold out a single test set per site (never used in training) or use leave-one-site-out for external validation.
Run multiple seeds (≥5) and report mean ± SD; bootstrap CIs for AUC and PR AUC.
Report per-site results and pooled (macro/micro) aggregates.
Ablations: vary communication rounds, local epochs, client fraction, and DP epsilons.

------------------------------------------------------------------------------------------------------
Statistical tests & uncertainty

AUC CI: DeLong or bootstrap.
Paired comparison: DeLong for paired AUCs; McNemar for paired classification at a threshold; bootstrap for differences in metrics.
Report p-values and effect sizes plus CIs.

------------------------------------------------------------------------------------------------------
Visualizations to include in thesis

ROC and PR curves with CIs (pooled and per-site).
Calibration plots (reliability diagram) and calibration tables.
Convergence plots: global validation metric vs rounds; show communication rounds on x-axis.
Bar/box plots of per-site AUCs (federated vs centralized vs local).
Heatmap of per-site performance differences.
Tradeoff plots: accuracy vs communication bytes; utility vs DP epsilon.
Decision curve (net benefit) vs threshold.

------------------------------------------------------------------------------------------------------
Implementation tips & tools

Use scikit-learn: roc_auc_score, precision_recall_curve, calibration_curve, brier_score_loss, confusion_matrix.
For CIs/bootstrap: resample test set or use sklearn.utils.resample and compute percentiles.
Logging: log per-round metrics (TensorBoard, MLflow, or simple CSV); keep per-client logs.
Federated frameworks: Flower/TFF/PySyft — instrument client/server hooks to capture communication bytes and timings.
Reproducibility: fix seeds, log dataset splits, and share requirements.txt and training commands.

------------------------------------------------------------------------------------------------------
What to report in thesis (minimal required)

Dataset: sites, sample counts, prevalence, inclusion/exclusion criteria.
Model: architecture, hyperparams, training protocol (local epochs, learning rate, aggregation).
FL setup: client sampling, rounds, aggregation algorithm, privacy settings.
Baselines compared and evaluation protocol (cross-validation or holdout).
Metrics with CIs, per-site breakdown, statistical tests, and figures.
Limitations: data heterogeneity, privacy assumptions, compute constraints.
