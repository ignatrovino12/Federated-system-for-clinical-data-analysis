Introduction

The rapid digitalization of healthcare systems has created an urgent need for platforms that can efficiently manage medical data while safeguarding patient privacy and security. Health clinics continuously generate vast amounts of sensitive information, including medical histories, diagnostic results, treatment records, and patient data. When properly managed and analyzed, this information can dramatically improve the early detection and prevention of chronic diseases such as diabetes and cancer.

However, traditional centralized systems face significant challenges in data privacy, scalability, and secure inter-institutional data sharing. While modern machine learning techniques offer powerful capabilities for analyzing medical data and identifying patterns that predict disease development, they typically require centralized data aggregation, which raises serious privacy concerns.

A promising solution for these issues is federated learning. Without requiring raw data to leave its local institution, this method allows machine learning models to be trained across several decentralized data sources. Federated learning achieves both better predictive performance and increased privacy protection by keeping sensitive patient data within each clinic while still enabling models to learn from collective knowledge across multiple sites.

Additionally, efficient communication between dispersed components of healthcare applications can be facilitated by contemporary web architectures like pub-sub systems. A stable environment for creating scalable and safe web platforms that can support such architectures is essential for this project.

The project presented in this work proposes a healthcare management platform based on a Django publish–subscribe architecture combined with federated learning techniques. The system aims to assist clinics in managing patient information while enabling predictive analysis of diseases such as diabetes. A key focus of the project is maintaining strong data security and protecting sensitive patient information while still enabling useful medical insights.

Existing technologies and solutions
The ongoing digitalization of healthcare systems has significantly increased the need for platforms capable of managing large volumes of sensitive medical data while simultaneously ensuring strict privacy and security guarantees. Healthcare institutions continuously generate heterogeneous data streams, including electronic health records (EHRs), laboratory results, imaging data, and data produced by wearable medical devices, and thus, effectively integrating and analyzing this information is essential for the early detection and monitoring of chronic diseases such as diabetes or cancer.  
However, traditional centralized data management systems present several limitations. These architectures often suffer from scalability bottlenecks, especially when dealing with large-scale distributed data sources. Additionally, centralized storage of sensitive medical information presents serious privacy concerns, particularly in Europe, where data protection regulations such as the General Data Protection Regulation (GDPR) and the Health Insurance Portability and Accountability Act (HIPAA) exist. These difficulties have prompted researchers to investigate decentralized methods that preserve institutional data privacy while enabling cooperative analysis of medical data.
To better understand the current technological landscape, I examined several recent research articles, focusing on decentralized learning systems, data virtualization frameworks, and distributed architectures designed for large-scale public health analytics. After examining these works, I found specific tendencies regarding the technological directions and design principles particularly relevant for the development of modern healthcare data platforms. In the following sub-chapters, these traits will be explained:
1. Fully Decentralized Edge Architectures & Scalability

Recent research trends have moved away from traditional classic centralized parameter server architectures toward fully decentralized learning models designed to support large networks of edge devices. This change is particularly relevant in healthcare environments where Internet of Medical Things (IoMT) devices, such as wearable sensors, continuously generate health-related data that must be processed efficiently and securely.

One notable example is the Totoro framework, which proposes a decentralized federated learning architecture based on a Distributed Hash Table (DHT) peer-to-peer communication model. Within this system, each healthcare application domain, such as diabetes prediction or cancer detection, is assigned its own dedicated parameter server structure. Unlike traditional federated learning systems, where a single centralized server coordinates the training process, this framework allows any edge node to dynamically transform into the coordinator or into the aggregator.

A key innovation introduced by this approach is the use of a publish-subscribe-based “forest” abstraction. In this model, each healthcare application operates within an independent dataflow tree that manages its own communication and training processes. This design significantly improves system scalability by ensuring that increased activity in one clinical application does not negatively affect the performance of other tasks running within the network.

To further address the challenges posed by unreliable clinical network connections, the framework incorporates a bandit-based path planning mechanism that dynamically routes communication through the most efficient and reliable network paths. This adaptive routing strategy improves overall training efficiency while maintaining robustness in environments with fluctuating connectivity.

2. Data Virtualization and Unified Access

Another big problem in healthcare information systems is the presence of semantic heterogeneity across different clinical data sources. Healthcare organizations often use a wide range of specialized systems, including electronic health record systems (EHRs), laboratory information systems (LIS), and IoMT platforms. These systems frequently rely on different data formats, schemas, and medical terminologies, making direct integration both complex and costly.

To fix this issue, modern healthcare architectures increasingly rely on data virtualization techniques, which provide a unified logical representation of distributed data without requiring physical data movement. Platforms such as Denodo, combined with event-driven messaging infrastructures like Google Pub/Sub, enable institutions to access heterogeneous clinical datasets through a single logical interface, and in this way, the systems remain autonomous. 

Data Virtualization significantly reduces integration complexity, with studies reporting reductions of up to 65% in development effort for data integration tasks. When combined with real-time publish-subscribe communication layers, these architectures also enable near real-time data availability. In certain implementations, data latency can be reduced from several minutes to less than one second, which is particularly valuable in scenarios that require fast clinical responses, such as real-time alerts for chronic disease monitoring.

3. Efficient Integration of Heterogeneous Features (VFL)

Healthcare data is often vertically partitioned, meaning that different institutions may possess different information and data describing the same patient. For example, one clinic may maintain detailed genetic data, while another may collect lifestyle information or long-term diagnostic records. Traditional machine learning approaches struggle in such environments because sharing raw patient data across institutions is typically restricted due to privacy regulations.

Vertical Federated Learning (VFL) is a solution for addressing this problem by enabling collaborative model training across institutions without requiring direct data sharing. However, conventional VFL implementations often suffer from high communication overhead and training delays due to the requirement for identity alignment, a process that ensures that records from different datasets correspond to the same individual.

Recent research has introduced PubSub-VFL, an architecture that integrates VFL with a publish–subscribe communication model. In this approach, identity alignment is decoupled from the main training process, allowing different institutions to process alignment tasks asynchronously. This design reduces synchronization constraints and allows training tasks to proceed more efficiently.

Additionally, the system introduces a hierarchical asynchronous training mechanism, which significantly improves performance in environments where participating institutions have heterogeneous computational resources. Experimental evaluations indicate that such architectures can reduce training latency by a factor of two to seven compared to traditional synchronous federated learning systems.

4. Autonomous "Model Streaming" and Knowledge Distillation

While federated learning enables collaborative training across institutions, it typically requires tightly coordinated interactions between participants. In real-world healthcare environments, however, institutions often have different analytical workflows, computational capabilities, and modeling preferences.

To address these limitations, research proposes the PubSub-ML framework, an alternative for federated learning, which introduces the concept of model streaming. In this approach, expert institutions, such as specialized research hospitals, act as publishers that stream differentially private machine learning models. Other institutions subscribe to these model streams and use them as “teacher” models to guide the training of local “student” models through knowledge distillation techniques.

This loosely coupled collaboration model allows institutions to maintain autonomy in their modeling approaches. For instance, one clinic may employ deep neural networks for disease prediction, while another may rely on classical machine learning methods such as random forests. Despite these differences, both systems can benefit from shared knowledge transmitted through model streaming.

The framework also addresses the issue of concept drift, which occurs when disease patterns evolve over time. By employing dynamic ensemble techniques and continuously updating streamed models, the system can adapt to changing clinical trends while maintaining a stable privacy budget for repeated updates.


5. Security, Privacy, and Public Health Policy

The integration of artificial intelligence technologies into healthcare systems requires strict compliance with regulatory frameworks and strong privacy guarantees. Medical data is highly sensitive, and improper handling can lead to severe ethical, legal, and social consequences.

Modern federated learning systems, therefore, incorporate privacy-by-design principles, integrating advanced privacy-preserving mechanisms directly into the training process. One commonly used approach is Gaussian Differential Privacy (GDP), which protects intermediate model representations, such as embeddings, from potential inversion attacks. In such attacks, adversaries attempt to reconstruct sensitive patient information from trained models or gradients.

Recent architectures further extend these mechanisms by introducing personalized privacy budgets, allowing each participating institution to define its own privacy parameters based on local regulatory requirements and risk tolerance. This flexibility is particularly important in international collaborations, where legal frameworks and data governance policies may vary significantly.

From a public health perspective, federated learning systems have shown considerable promise. Several analytical reviews indicate that federated models can achieve improved accuracy and fairness when predicting disease risk, classifying cancer types, or detecting infectious diseases such as COVID-19. By enabling models to learn from diverse patient populations without compromising data sovereignty, federated learning represents a good solution for large-scale, privacy-preserving medical analytics.

Proposed System Architecture and Technology Selection

Building upon the technologies and approaches discussed in the previous sections, the developed project proposes a distributed healthcare analytics platform designed to support secure data management and collaborative predictive modeling across multiple clinical institutions. The proposed system combines an event-driven publish-subscribe communication architecture with federated learning mechanisms, enabling healthcare providers to collaboratively train predictive models while preserving the privacy of sensitive patient data.

The architecture is designed to address several of the challenges identified in current healthcare information systems, including data silos, limited interoperability between clinical systems, strict privacy constraints, and the need for scalable data processing infrastructures. By integrating distributed messaging systems, asynchronous task execution, and federated learning coordination, the platform aims to support efficient collaboration between healthcare providers while maintaining local control over patient data. Now I will discuss the principal components of the described system:



1. Event-Driven Communication Through Publish–Subscribe Architectures

The central component of the proposed system is the adoption of an event-driven communication model based on the publish-subscribe paradigm. In distributed healthcare environments, traditional request-response architectures often introduce tight coupling between system components and can lead to scalability limitations when multiple institutions participate in collaborative workflows.

To address these limitations, the proposed architecture employs a message broker implemented using Redis to support a pub-sub messaging infrastructure. Within this model, system components communicate indirectly through event channels rather than through direct service calls. Producers publish messages describing events or tasks, while consumers subscribe to relevant channels and process these events asynchronously.

This design provides several advantages regarding distributed healthcare systems. First, it enables loose coupling between system components, allowing clinics to participate in collaborative processes without requiring direct dependencies between their systems. Second, the asynchronous nature of the messaging architecture improves system scalability, as new participants can be integrated without significantly affecting the communication patterns of existing components. Finally, the event-driven model increases system resilience, since temporary service disruptions do not necessarily interrupt the entire workflow.

Within the proposed platform, the publish-subscribe infrastructure coordinates several key operations, including task distribution, model update propagation, and synchronization between participating clinical nodes.

2. Distributed Clinical Nodes and Local Data Management

Each participating healthcare institution operates a local system node responsible for managing patient data and participating in collaborative model training. These nodes are implemented as web applications built using the Python Django framework, which provides a robust and secure medium for developing data-driven web services and interfaces.

Patient information is stored locally using PostgreSQL databases. Maintaining local data storage is an essential requirement for healthcare environments, as medical records are subject to strict privacy regulations and institutional governance policies. By ensuring that patient data remains within the infrastructure of the originating clinic, the architecture preserves data sovereignty while still permitting collaborative analysis through federated learning mechanisms.

In addition to data management functionalities, the local nodes are responsible for executing machine learning tasks on locally available datasets. These tasks may include model training, feature extraction, and prediction generation for diseases. The results of these processes contribute to the global learning process without exposing raw patient data.

3. Asynchronous Task Processing

Given the computational complexity associated with data processing and machine learning training, the architecture incorporates asynchronous task execution mechanisms to prevent blocking operations within the main application services. Background task processing is implemented using distributed worker processes based on Celery.

Celery workers subscribe to task queues managed by the Redis broker and execute operations such as local model training, data preprocessing, and communication with the federated learning coordinator. This approach improves the responsiveness of the web application while enabling the system to handle computationally intensive tasks efficiently.

Furthermore, asynchronous task processing allows the system to better accommodate heterogeneous computing environments, as participating clinics may have different hardware capabilities or workload constraints.

4. Federated Learning Coordination

To enable collaborative predictive modeling across institutions without requiring direct data sharing, the proposed system integrates federated learning mechanisms coordinated through the Flower Framework. Federated learning enables multiple participants to train a shared machine learning model by exchanging model parameters rather than raw datasets.

In the proposed architecture, each clinic trains a local model using its own dataset. Periodically, the model updates generated locally are transmitted to the federated learning coordinator, which aggregates them to produce a global model representing the collective knowledge of all participating institutions.

The aggregated global model is stored and distributed through object storage implemented with MinIO. This is an S3-compatible storage system that allows participating nodes to retrieve updated model versions while maintaining a centralized yet secure repository for trained models.

By adopting this approach, the system ensures that sensitive medical information remains within the local infrastructure of each clinic while still benefiting from collaborative learning across institutions.

5. Containerized Deployment and System Scalability

To facilitate deployment across heterogeneous infrastructures, the system components are packaged as containers using Docker. Containerization ensures consistent execution environments and simplifies the distribution of application components across different computing platforms.

For large-scale deployments involving multiple clinical institutions, container orchestration is managed using Kubernetes. Kubernetes provides automated service management, resource allocation, and fault tolerance mechanisms that enable the platform to scale dynamically as the number of participating clinics increases.

This infrastructure design allows the proposed system to support geographically distributed healthcare networks while maintaining high availability and operational stability.

6. Monitoring and System Observability

Because the architecture relies on multiple distributed components, monitoring mechanisms are necessary to ensure reliable system operation. The proposed platform integrates monitoring and observability tools such as Prometheus for metrics collection and Grafana for visualization.

These tools make it possible to monitor system performance, resource utilization, and federated learning activity in real time. Continuous monitoring helps identify performance bottlenecks, detect anomalies, and ensure that distributed services operate correctly within the collaborative healthcare network.
